Technical Design Document for Image Processing System

1. Introduction

The Image Processing System processes image data from CSV files. It allows users to upload a CSV file containing product information and image URLs, processes the images asynchronously (compression by 50%), and provides a request ID for tracking progress. Users can later download a CSV with processed image URLs.

2. System Architecture

The system consists of the following components:

2.1 Components

Frontend (React/Vite): Provides a UI for users to upload CSV files and download processed results.

Backend (Node.js, Express.js): Handles CSV validation, image processing, and database operations.

Database (MongoDB): Stores request metadata, product details, and image URLs.

Storage (Cloud/Local): Stores processed images.

Worker Service: Asynchronously processes images.

Webhook Service (Optional): Notifies external systems after processing.

3. Workflow

3.1 CSV Upload Process

User uploads a CSV file.

API validates the CSV format.

Make sure that the CSV to be uploaded must have the headers columns names as below to get validated:

S. No. | Product Name | Input Image Urls

A unique request ID is generated and stored in the database.

The request is queued for asynchronous processing.

3.2 Image Processing

The worker service fetches queued image processing tasks.

Each image is downloaded, compressed by 50%, and stored.

The database is updated with processed image URLs.

3.3 Status Query

The user requests status using the request ID.

The API checks processing status and returns it.

3.4 CSV Download

Once processing is complete, the system generates an output CSV.

The user downloads the CSV with input and output image URLs.

4. API Endpoints

4.1 Upload CSV

Endpoint: POST /api/upload-csv

Request: Multipart file (CSV)

Response: { request_id: "abc123" }

4.2 Check Request Status

Endpoint: GET /api/request-status/:requestId

Response: { status: "COMPLETE" }    --  PENDING as default. FAILED when error occured during the compression.

4.3 Download Processed CSV

Endpoint: GET /api/export-csv/:requestId

Response: CSV file with processed images

5. Database Schema

5.1 Request Collection

{
  "request_id": "abc123",
  "status": "PENDING",
  "created_at": "",
  "updated_at": "",
}

5.2 Product Collection

{
    "request_id": "abc123",
    "product_name": "SKU1",
    "input_images_urls": ["url1", "url2"],
    "output_images": ["processed_url1", "processed_url2"],
    status: "COMPLETE",
    "created_at": "",
    "updated_at": ""
}

6. Technologies Used

Frontend: React.js, Vite, Tailwind CSS

Backend: Node.js, Express.js

Database: MongoDB

Image Processing: Sharp (for compression)

Deployment: Vercel (Frontend), Render (Backend)

7. Error Handling & Edge Cases

Invalid CSV Format: Reject upload with a descriptive error message.

Missing Images: Log missing URLs and continue processing.

Slow Processing: Implement retries for failed tasks.
